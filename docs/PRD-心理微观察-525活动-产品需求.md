## 1. 背景与目标

### 1.1 背景

在 **525 心理健康节**节点，通过「微颗粒」隐喻与轻量治愈系视觉，引导用户完成**测评 → 猜颗粒 → 读报告**，传递自我觉察与关系视角下的行为洞察。

### 1.2 产品目标

| 目标 | 说明 |
|------|------|
| **完成率** | 单步单屏、路径清晰，减少中途跳出。 |
| **价值感知** | 报告分层展示（自我 / 他人视角 / 建议）；**猜中动物人格**时强化「自我性格分析免费」的正向反馈。 |
| **传播与裂变** | 活动链接拉新、报告模块图分享促传播；**邀请好友成团**经微信 → 小程序发券，双向激励。 |
| **商业化** | 全站付费入口统一走 **测测收银台（测测币）**，与 App 充值体验一致。 |

### 1.3 成功指标（建议正式环境量化）

- 活动 UV → 测评完成率、报告页到达率、付费点击率、分享次数、成团链接点击率、券领取与核销率。

### 1.4 非目标

- 量表与动物算法的临床效度论证（对外表述需法务/学术审核）。
- **H5 内嵌**完整收银台 UI（由 **测测 App 原生**承接；H5 仅调起协议与参数）。

## 2. 范围

### 2.1 In Scope（产品规格）

- 落地页 → 身份 → 70 题 → 猜颗粒 → 报告（含猜中/未猜中策略）。
- **分享（三类）**：活动链接拉 App；报告**分模块**生成图片分享；成团链接 → 微信 → 小程序领 7 折券（被邀请人与分享者规则）。
- **付费**：所有解锁/付费类按钮统一拉起 **测测收银台**（测测币充值界面，见第 7 章）。

### 2.2 Out of Scope（实现边界）

- H5 内完整自绘收银台（非 App 原生充值页）。
- 测测币汇率、支付渠道签约细节（以支付中台为准）。

## 3. 用户与场景

### 3.1 角色

| 角色 | 描述 |
|------|------|
| **测评用户** | 完成测评与报告阅读；可能付费解锁、分享活动或报告模块。 |
| **被邀请用户** | 点击微信内成团链接，进入小程序，符合条件则领取 7 折券并被引导回 App。 |
| **分享者** | 发起成团分享；成功规则满足时自身也获得 7 折券（与业务规则一致）。 |
| **运营/产品** | 配置价格常量、活动页链接、券规则、SKU 与测测币标价。 |

### 3.2 核心用户故事

1. **As a** 用户，**I want** 在明确身份语境下完成测评，**so that** 报告中的「他人眼中的你」能贴近真实关系场景。  
2. **As a** 用户，**I want** 用直觉猜动物人格并看到是否猜中，**so that** 获得「惊喜 + 权益差异」的感知。  
3. **As a** 用户，**I want** 在付费前看清报告结构与锁定范围，**so that** 决策是否解锁完整内容。  
4. **As a** 用户，**I want** 分享整个活动或单块报告图为图片，**so that** 好友能一键进入测评或看到我的结果片段。  
5. **As a** 用户，**I want** 付费时与 App 内充值体验一致，**so that** 我清楚余额与需充值金额。  
6. **As a** 分享者，**I want** 好友通过链接领券后我也能获得券，**so that** 有动力持续邀请。

## 4. 整体业务逻辑方案（核心）

### 4.1 主流程（顺序固定）

```text
[Landing 落地页] ——顶栏可分享活动链接（类型 1）
    → 主 CTA「开始测试」
[Identity 身份定频] ——必选 1 个身份
    → 「开始 70 题采样」
[Quiz 答题] ——70 题 Likert，逐题提交
    → 答完最后一题自动进入
[Guess 猜颗粒] ——6 张动物卡片选 1，确认提交
    → 系统判定是否猜中 + 计算匹配度
        ├─ 猜中 → 「认知天才」全屏反馈（可跳过）→ 报告
        └─ 未猜中 → 直接进入报告
[Report 报告] ——三大块 + 各模块分享（类型 2）+ 底部付费（收银台）与成团（类型 3）
```

### 4.2 状态机要点（完整）

| 节点 | 关键规则 |
|------|----------|
| **身份** | 未选择身份则「开始 70 题采样」不可用。 |
| **答题** | 选中 Likert 即提交本题并进入下一题；**「开始采样」会重置**答题进度；从答题返回身份**不自动清空**已答进度（再次点「开始采样」会重置）。 |
| **猜颗粒** | 真实动物由 `pickTrueAnimal(profile)` 在 **16** 只动物中选定；选项池 = **1 真 + 随机 5 只**，共 **6** 张，洗牌展示。 |
| **猜中判定** | 用户所选 `id` === `trueAnimal.id` → `guessExact = true`，`matchPercent = 100`。 |
| **未猜中** | `guessExact = false`，`matchPercent` = 所选动物向量与用户 profile 的**余弦相似度**（0～100 整数）。 |

### 4.3 报告解锁策略（产品规则）

| 条件 | 第一部分「自我性格分析」 | 第二部分「他人眼中的你」 | 第三部分「后续建议」 | 底部主按钮锚定价格 |
|------|--------------------------|---------------------------|----------------------|----------------------|
| **猜中** | **全文可见**（无自我区锁层） | 磨砂预览 + 需付费解锁 | 锁定预览 + 需付费解锁 | **¥12.9** 解锁「他人 + 建议」（`PRICE_OTHERS`） |
| **未猜中** | **锁定预览**（底部带状蒙层 + 正文模糊；**模块大标题「自我性格分析」保持清晰**） | 同上 | 同上 | **¥19.9** 解锁完整报告（`PRICE_FULL`） |

**价格常量（前端演示，正式以后台为准）**

- `PRICE_FULL = 19.9`（完整报告）
- `PRICE_OTHERS = 12.9`（仅他人视角 + 后续建议，猜中后）
- `DISCOUNT = 0.7`（成团折扣系数，用于「人均约 ¥xx」）

**付费形态（正式环境）**：标价可与 **测测币 1:1** 展示；用户点击付费后，**差额**通过收银台充值完成（见第 7 章）。

### 4.4 与 UI 呈现相关的逻辑

- **落地页**：治愈系浅绿主题；插画区 **525 · 心理健康节**；主标题 **「RTC微颗粒心理测评」**；副标题强调关系场景与沟通价值；底部背书含 **RtCatch 已累积样本量超过 2,300 万人次**（演示数据以产品为准）。  
- **结果页**：**不展示**顶部「匹配度 / 猜中」横幅。  
- **未猜中自我区蒙层**：**底部带状**霜层，与「他人眼中的你」风格一致，便于理解结构。  
- **演示版**：付费可为占位弹窗；正式版调起 **测测收银台**。

## 5. 信息架构与页面流

整体为 **5 个视图**，通过 `showView(viewName)` 切换；**落地页不展示**顶部四步导航，进入「身份」后展示。

| 顺序 | view 名 | 用户可见名称（顶栏） | 说明 |
|------|-----------|----------------------|------|
| 1 | `landing` | （无四步导航） | 品牌、活动分享、主 CTA；星云等动效以实际实现为准 |
| 2 | `identity` | 身份 | 选择测评所锚定的身份 |
| 3 | `quiz` | 答题 | 70 题 Likert，进度环 |
| 4 | `guess` | 猜颗粒 | 6 选 1；强调惊喜与猜中免费自我分析（**本页不展开具体付费金额**） |
| 5 | `report` | 报告 | 三大块 + 分模块分享 + 底部付费坞 |

**导航规则**

- **身份页**：可返回落地页。  
- **答题页**：可返回身份页；**重选身份后再次「开始采样」会重置答题进度**。  
- **猜颗粒、报告**：当前无「返回上一步」按钮（若产品需要单独立项）。

## 6. 功能需求（分页面）

### 6.1 落地页（Landing）

- **顶栏**：品牌「测测 × RTC」；**分享**按钮 —— **类型 1：整体活动链接**（带渠道参数；点击后分享，好友打开可 **拉起/下载测测** 进入活动，正式环境用 Universal Link / 应用商店兜底）。  
- **主视觉**：插画 + **525 · 心理健康节** 标签。  
- **主标题 / 副标题**：RTC微颗粒心理测评；关系场景与沟通价值。  
- **主 CTA**：「开始测试」→ 身份页；可配合 `MicroAudio` 环境音（以实际为准）。  
- **信任与隐私**：量表与算法背书、样本量（演示数据）、结果加密说明。  
- **动效**：星云 Canvas 等（非落地页可停止以省资源，以实际为准）。

### 6.2 身份定频（Identity）

**必选**：须选择一种身份后方可「开始 70 题采样」。

**身份列表（当前实现）**

| id | 展示名 | 场景说明（卡片主文案） |
|----|--------|------------------------|
| girlfriend / boyfriend | 女朋友 / 男朋友 | 恋爱关系中的心理状态评估；选中时展示双人剪影装饰 |
| wife / husband | 妻子 / 丈夫 | 婚姻关系中的心理健康分析；双人剪影装饰 |
| mother / father | 母亲 / 父亲 | 亲子关系中的心理状态了解 |
| worker | 工作者 | 职场压力与心理健康评估（合并原基层/管理者场景） |
| student | 学生 | 学业压力与心理状态分析 |

每条身份含：**场景说明**（`reason`）、一句**钩子**（`hook`）；选中后高亮并写入 `--identity-accent` 色。

**主按钮**：「开始 70 题采样」；未选身份时禁用。

### 6.3 答题（Quiz）

**题量与题型**

- 共 **70** 题（`MICRO_QUESTIONS.length`），单选 **1～5 分** Likert（非常不同意 → 非常同意）。  
- 部分题目 **反向计分**（`inv`），在 `recordAnswer` 中按 `6 - score` 处理。

**维度（五维）**

- 自我觉察、人际亲和、情绪稳定、规则计划、开放探索  

每题归属 `dim`，累加分数与题数，答完后归一化为 **0～100** 的 `profile[5]`。

**进度与反馈**

- 当前题号 / 总题数、环形进度百分比。  
- 答题过程显微镜模糊度随进度变化（`--lens-blur`）。  
- 在第 **20、40、60** 题完成后触发 Toast 鼓励语，约 **3.2s** 消失。

**操作**

- 点击某一 Likert 选项即 **提交本题并进入下一题**（无单独「下一题」按钮）。

### 6.4 猜颗粒（Guess）

**业务叙事**

- 强调惊喜：若与系统判定一致，**「自我性格分析」全文免费**。  
- **本页不展示**具体付费档位、成团细则（归属报告页与后端）。

**算法与选项**

- `pickTrueAnimal` 在 16 只动物中与用户向量余弦相似度最高者为真实结果。  
- 选项池：**1 真 + 从其余 15 只随机 5 只**，共 6 张，洗牌。  
- 每张卡片：emoji、名称、一句话描述。

**交互**

- 点选一张为选中态；「确认选择」提交。  
- **猜中**：「认知天才」全屏层，匹配度 100%，约 **2.2s** 自动进报告，或点「查看报告」。  
- **未猜中**：直接进入报告；匹配度为所选动物与 profile 的余弦相似度（0～100%）。

### 6.5 报告（Report）

**三部分结构**

1. **自我性格分析**：动物人格、雷达、心理健康指数、特质、解析、压力关注点等。  
2. **他人眼中的你**：基于所选身份的四模块叙事；默认 **磨砂遮罩 + 解锁**。  
3. **后续建议**：正文 + 锁定区与解锁入口。

**猜中 / 未猜中**与价格：见 **§4.3**。

**分享（类型 2）**

- **每一模块标题行右侧**设 **分享** 按钮（`aria-label` 如「分享本模块为图片」）。  
- 点击后对 **当前模块对应区块 DOM** 生成图片（实现可采用 html2canvas / 原生截图 / 服务端合成），再调起系统分享或保存；**仅截取当前已展示内容**，避免误导。  
- **不再**以整页单一「生成分享卡片」大按钮为主入口（可与分模块方案短期并存，**验收以分模块为准**）。

**付费与成团**

- **底部付费坞**：`reportPayTip`、主按钮 `btnPayPrimary`、次按钮「邀请好友成团 · **共享 7 折**」、成团说明 `reportPayGroupHint`。  
- **模块内** `btnUnlockOther`、`btnUnlockAdvice` 与主按钮一致走 **收银台**（SKU 区分，见第 7 章）。  
- **类型 3（成团）**：次按钮分享至 **微信会话**，打开 **小程序** 领券与回 App 逻辑见 **§8.3**。

## 7. 付费模块 — 测测收银台（统一入口）

### 7.1 原则

- **所有付费相关按钮**在正式环境均 **拉起测测 App 内收银台**（与「充值」一致）。  
- **WebView 内嵌**：`JSBridge` / `CeceBridge.openCashier`（命名以实际 SDK 为准）。  
- **系统浏览器**：Deeplink 唤起 App 或下载引导。

### 7.2 收银台界面结构（与 App 现网一致）

| 区域 | 说明 |
|------|------|
| **标题** | 「充值」 |
| **面额网格** | 3×3 共 9 档 **测测币（C）**：如 1、4、10、30、50、100、200、500、1000（以后台为准）。 |
| **单格** | 金色 **C** 图标 + **币数量** + 下方 **人民币价格（1:1）**。 |
| **选中态** | 高亮边框 / 浅粉背景等。 |
| **余额** | 左侧 `C xx.xx`。 |
| **需充值** | 右侧为 **应付 − 余额** 差额，动态变化。 |
| **支付** | **支付宝充值**、**微信充值** 并排。 |
| **合规** | 勾选「阅读并同意《测测币充值协议》」。 |
| **客服** | 「充值遇到问题？立即反馈」。 |

### 7.3 业务参数对接（建议）

| 参数 | 说明 |
|------|------|
| 活动 / SKU | `psych_report_full` / `psych_others_advice` 等 |
| 应付金额 | 人民币或测测币；收银台按差额推荐档位 |
| 回调 | 支付成功 → 关闭收银台 → 解锁对应报告权益 |

### 7.4 演示版

- 无 Bridge 时 **弹窗说明**将拉起收银台及 SKU、金额；联调后替换为真实调用。

## 10. 非功能需求（摘要）

- **布局**：单屏视图、主区可滚动、报告底栏固定、**safe-area**。  
- **无障碍**：猜颗粒 `listbox` / `option`、`aria-selected`（可继续加强）。  
- **性能**：非落地页可停星云；雷达为 Canvas；**分模块截图**注意内存与失败 Toast。  
- **收银台调起**：超时、失败回退与客服入口。

## 11. 验收标准（示例检查清单）

### 11.1 主流程

- [ ] 身份 / 答题 / 猜颗粒 / 报告顺序与解锁规则与 **§4.3** 一致。  
- [ ] 未猜中自我区：标题清晰、正文锁定形态正确。  
- [ ] 无顶部匹配度横幅。

### 11.2 分享

- [ ] 类型 1：分享链接可尝试拉起 App（正式）；演示可 Web Share / 复制。  
- [ ] 类型 2：三模块各有分享，生图内容对应当前模块。  
- [ ] 类型 3：微信分享链路、小程序与券逻辑与运营规则一致。

### 11.3 付费

- [ ] 所有付费入口调起 **测测收银台**（或 Bridge 占位可验收）。  
- [ ] 收银台含面额、余额、需充值、协议、双渠道支付（App 内）。

## 12. 与后端 / 原生对接清单（研发）

1. 测评与报告存证、防篡改重算。  
2. 订单与 SKU、测测币扣减与回调。  
3. **Bridge**：`openCeceCashier({ sku, amountYuan, biz, title })` 或现网等价接口。  
4. 分享：活动链接、小程序 path、邀请关系、券发放。  
5. 类型 2 生图可选服务端合成。

## 13. 开放问题

- 从答题返回身份是否需「退出测评」二次确认。  
- 猜颗粒 / 报告是否提供返回上一步。  
- 类型 3：分享者获券 **触发条件**（即时 vs 好友付费后）。  
- 对外样本量、量表名称终稿（法务）。  
- 收银台是否支持 **自定义充值金额**（与九宫格并存）。
